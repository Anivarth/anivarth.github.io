Why and What and How is Travis CI

Published: Sun 11 February 2018
By PSPT

In TheSciPost.

tags: CI documentation

What is Continuous Integration?

It is to make sure that code doesn't break at any point of time. The paradigm says to commit the code as frequently as possible to avoid 'merge hell'. These atomic commits along with unit testing will help prevent a lot of problems in the future.

What is Travis CI?

It helps in testing commits from many developers. Lets say there are many persons working on your code. A new person commits a new code without testing his version. If the new code breaks then the Travis CI will show the error and help us to sort out the issue.

I faced this issue when I tried adding new code to pelican theme.

How to Travis CI?

It is very straight forward. The following are required: - version control - tests.py file - .travis.yml file

My example of tests.py file:

...code:: python

import unittest

def test_one_plus_one_two:
return assertEqual(1+1, 2)
if __name__ == "__main__":
unittest.main()

... My example of .travis.yml file:

...code::yaml

langauge: python python:

  • 2.7
  • 3.6

script: python tests.py

...

Thats it. You can now push it to github.

Travis CI will take care of all your requirements.
